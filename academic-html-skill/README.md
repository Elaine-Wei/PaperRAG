# Academic HTML Article Skill — 使用指南
 
## 这是什么？
 
一个 Claude Skill，让 Claude 生成**统一风格**的学术教程 HTML 文件。效果是白底、衬线标题、KaTeX 数学公式、彩色 callout 框、定理块、SVG 图解的排版风格——类似高质量的论文精读讲义。
 
---
 
## 安装（一次性操作）
 
1. 拿到 `academic-html-article.skill` 文件
2. 在 Claude.ai 中开一个**新对话**
3. 把 `.skill` 文件拖进聊天框上传
4. Claude 会提示安装，确认即可
安装成功后，所有**新对话**都可以使用该 skill（已有对话不受影响）。
 
> **注意**：Skill 需要 Claude.ai 的 **"Code Execution and File Creation"** 功能开启（Settings → Feature Previews），否则 Claude 无法生成文件。
 
---
 
## 两种使用方式
 
### 方式一：聊完论文后导出 HTML
 
这是最常见的用法。你和 Claude 讨论一篇论文或技术话题，讨论完之后让它整理成 HTML。
 
**典型流程：**
 
```
你：[上传论文 PDF 或贴论文链接]
你：帮我讲讲这篇论文的核心思想
Claude：[详细讲解...]
你：那个定理的证明能再展开说说吗？
Claude：[展开证明...]
你：好的，帮我把刚才讨论的内容生成HTML
Claude：[调用 skill，生成带完整排版的 .html 文件]
```
 
**触发关键词（说以下任何一句都会触发）：**
 
- `帮我生成HTML`
- `导出成HTML`
- `帮我整理成HTML文件`
- `generate HTML` / `export to HTML`
**效果**：Claude 会回顾整个对话，提取讨论过的概念、公式、例子，组织成一篇结构化的文章，套用模板风格输出 `.html` 文件。
 
---
 
### 方式二：上传已有文件转换格式
 
你已经有一份笔记（markdown、Word、PDF、或其他风格的 HTML），想转成这个模板的风格。
 
**典型 prompt：**
 
```
你：[上传 notes.md]
你：帮我把这个笔记转成HTML模板格式
```
 
```
你：[上传 old_article.html]
你：用这个风格重新排版
```
 
**支持的输入格式**：`.md`、`.txt`、`.html`、`.docx`、`.pdf`

## 生成的 HTML 有哪些组件？
 
| 组件 | 用途 | 什么时候出现 |
|------|------|-------------|
| **Header** | 标题、副标题钩子、论文引用 | 每篇都有，论文引用按需 |
| **目录 (TOC)** | 带序号的章节链接 | 3+ 章节时自动生成 |
| **Section (h2/h3)** | 主章节 / 子章节 | 内容的骨架 |
| **Callout Box** | 重点提示、警告、信息补充 | 关键概念、易错点、重要区别 |
| **Theorem / Proof** | 定理陈述 + 证明 | 有形式化定理时 |
| **Steps** | 带圆形序号的步骤列表 | 推导步骤、操作流程 |
| **Table** | 对比表格 | 概念对比、方法比较 |
| **Algorithm** | 伪代码块 | 算法描述 |
| **SVG Diagram** | 内嵌矢量图解 | 需要可视化解释时 |
| **Tags** | 行内彩色标签 | 标记对立概念（如 开环/闭环） |
| **KaTeX Math** | 行内 `$...$` 和独立行 `$$...$$` | 有数学公式时 |
 
### Callout Box 的四种颜色
 
- 🔴 **danger (红色)** — 关键区别、常见错误、陷阱
- 🟢 **success (绿色)** — 核心总结、关键结论
- 🔵 **info (蓝色)** — 补充说明、重要上下文
- ⚪ **default (中性)** — 一般注释

## 三种文章类型
 
Claude 会根据内容自动选择结构，你也可以指定：
 
| 类型 | 适用场景 | 示例 prompt |
|------|---------|------------|
| **论文精读** | 讨论一篇具体论文 | "把我们关于 DAgger 论文的讨论整理成 HTML" |
| **项目分析** | 分析代码库/系统设计 | "把 Vibe-Trading 的源码分析导出成 HTML" |
| **概念讲解** | 解释某个技术概念 | "把刚才讲的 Transformer Attention 机制生成 HTML" |

## 提高输出质量的技巧
 
### 1. 聊天阶段多问几轮再导出
 
Skill 从对话上下文提取内容——讨论越深入，生成的文章越充实。建议至少聊 5-10 轮再说"生成HTML"。
 
### 2. 可以指定重点
 
```
帮我生成HTML，重点放在 Theorem 2.1 的证明过程和直觉解释上
```
 
### 3. 可以要求加图
 
```
生成HTML，在"误差累积"部分加一个 SVG 图解
```
 
### 4. 生成后可以迭代
 
```
第3节太短了，把 XXX 的细节补充进去重新生成
```
 
### 5. 指定文章类型
 
```
用"项目分析"的结构帮我生成HTML
```

## 常见问题
 
**Q: 生成的文件怎么打开？**
A: 下载 `.html` 文件后用任意浏览器打开。公式渲染需要联网（KaTeX 从 CDN 加载），但正文和样式都内嵌在文件里。
 
**Q: 公式显示不出来？**
A: 需要联网。KaTeX 的 CSS 和 JS 从 `cdn.jsdelivr.net` 加载，确保网络可以访问。
 
**Q: 可以自己改生成后的 HTML 吗？**
A: 可以。文件就是标准 HTML，用任何文本编辑器打开修改即可。CSS 在 `<style>` 标签里，内容在 `<body>` 里，结构很清晰。
 
**Q: 中英文混排会不会排版错乱？**
A: 不会。模板已经配置了 Noto Sans SC / Noto Serif SC 作为中文回退字体。
 
**Q: 能不能只生成某几个章节？**
A: 可以。直接告诉 Claude 你要哪些部分，比如"只生成第 3-5 节的内容"。
 
**Q: Skill 安装后怎么确认生效了？**
A: 在新对话中随便说"帮我生成一个关于XXX的HTML"，如果 Claude 生成了带有 serif 标题 + callout box 风格的 HTML 文件就说明生效了。
