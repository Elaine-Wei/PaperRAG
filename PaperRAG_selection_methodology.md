# PaperRAG — 选择与质量过滤方法论 (Selection & Quality-Filtering Methodology)

**用途**：本文件记录 PaperRAG 论文遴选系统的**完整选择/质量过滤逻辑**，作为一个**可借鉴的参考框架**（不是照抄的规格），用于一个独立任务：构建一份 **30–40 个高质量 YouTube 频道/创作者白名单**。

**领域范围（与论文流水线一致）**：AI、Quant（量化）、Math（数学）、AI4Science（尤其 AI4Math）、CS exploration（计算机前沿探索）。

**核心理念一句话**：*多层召回保证覆盖 → 便宜的语义 LLM 判断替代脆弱的字符串匹配 → 多维打分（含权威性）而非单一热度 → 综评排序 → 「经典 × 最新」双轨呈现，让时间沉淀与新鲜前沿都不被对方压垮。*

流水线总览：

```
① 三层检索(seeds + keyword nets + broad) 
   → 便宜预筛(category/关键词，省 LLM 调用)
   → ② LLM 判官(单次调用：keep/drop + 分类 + 理由)
   → ③ 多维打分(freshness / reproducibility / novelty / domain-relevance / authority)
   → 综评 composite(单次 LLM，0–10，不改子分)
   → ④ freshness 双轨(steep/relaxed) + 奠基之作单列
   → ⑤ 双区呈现(经典按质量 / 最新按新鲜)
```

## 1. 候选来源：三层检索 (Candidate Sourcing — 3-Layer Retrieval)

候选池由三层合并、去重、加时间窗构成。三层各司其职，缺一不可。

### (a) 已验证种子锚点 (Verified Seed Anchors) — 按精确 ID 取 + 标题校验
- 人工挑选该领域**公认的经典/奠基之作**，以其 arXiv ID 直接抓取。
- **标题校验**：抓回后比对标题是否含预期关键词；不符 → 告警并回退到关键词搜索（`discard + keyword-fallback`），避免 ID 漂移取错论文。
- 种子**不受时间窗限制**（老经典即使超出「近两年」也保留）。
- 种子**跳过过滤**，但仍过一遍分类做 **sanity check（report-and-keep）**：分类器若与人工归属分歧，只**报告不剔除**（人工判断优先）。

### (b) 关键词网 (Keyword Nets) — 围绕每个种子的「谱系/延伸」
- 为每个种子 + 每个子方向布置一组关键词查询（相关度排序、每查询取较多结果，如 50）。
- 目的：抓住种子的**引用/延续/同门工作**——同一思路的后续论文、改进、变体。

### (c) 宽查询 (Broad Queries) — 捕捉「漏网的新东西」
- 更宽的领域查询，捕捉**不在已知谱系里的新工作**（新团队、新命名、跨界）。

### 为什么是三层（设计理由）
- **种子**保证经典**绝不遗漏**（哪怕检索算法抽风，钦定的必读仍在池中）。
- **关键词网**捕捉每个种子的**血脉延伸**（把一条研究线的上下游都网住）。
- **宽查询**兜住**新颖/意外**的工作（种子和关键词都想不到的东西）。
- 三者**合并去重**：`检索源不决定最终分类/去留`——来源只负责「进池」，质量与归类交给后续 LLM 判断与打分。

> 实现参考：`run_boards.retrieve_all()` / `run_topic.retrieve_topic()`（seeds by ID + KW nets + broad，dedup + 近 N 年时间窗，验证锚点豁免时间窗）。

## 2. LLM 语义过滤/分类 (LLM-Based Filtering & Classification)

**我们用一次便宜的 LLM 判官调用，取代了脆弱的正则/关键词闸门。**

### 演进背景（为什么换掉正则）
早期用正则主题闸（标题/摘要必须命中某些词）过滤跑题结果。问题：
- **误杀**（false-kill）：措辞不同的真命中被漏掉（正则太严 → 召回崩到每类 ~2 篇）。
- **误纳**（false-include）：物理里的 "alpha 粒子"、天文里的 "co-evolution" 混进来（正则太松 → 一堆噪声）。
- 正则只会**匹配字符串**，不懂**语义**。

### 现方案：预筛 + 单次 LLM 判官
1. **便宜预筛（省调用）**：先用极轻的规则砍掉明显无关的——「属于目标学科分类（如 cs.\*/q-fin.\*）**或**标题/摘要出现领域词」。物理的 "alpha 粒子"不含金融/agent 词 → 预筛即淘汰，不进 LLM。
2. **单次 LLM 判官（每候选一次）**：把 `标题 + 摘要(截断~1400字)` 交给一个便宜、快的模型，一次调用**同时完成过滤 + 分类 + 给理由**。
3. 丢弃 `keep=false`；保留者带上**分类**与**一句理由**进入打分。
4. 判官带 **503/限流退避重试**；解析失败不伪造结果。

### 判官 Prompt 结构（实际形态）
```
System（判定 + 分类 + 排除项 + 严格 JSON 输出）:
  "判断这篇是否属于【<目标主题的精确定义 + 机制清单>】。
   若是 → keep（并给出子类，如 A/B 或单一 keep）；否则 → drop。
   注意：<列出常见误命中，明确排除>（如：物理 alpha 粒子、纯基准/综述而非系统、
        天文/生物里的 co-evolution 等一律 false）。
   只输出 JSON：{\"is_<topic>\": true/false, \"<category>\": \"...\"/null, \"reason\": \"一句话\"}"

User:
  "标题：{title}\n摘要：{abstract[:1400]}"

模型：便宜快的模型；temperature=0；带退避重试。
```

### 为什么 LLM 判官优于关键词匹配
- **懂语义不懂字符串**：判断的是「这篇**在做**什么」，而非「出现了哪个词」。
- **同时过滤 + 归类 + 解释**：一次调用产出 keep/drop、类别、可复核的理由（理由让人能审出误杀/误纳）。
- **可分级留证**：把 before/after 数量与每篇理由打出来，人工可抽查判官质量（我们实测：281 候选 → 预筛 278 → 判官 keep 109 / drop 169，drop 理由逐条正确剔除了综述/基准/安全攻击/物理天文同名词/通用 RL）。

> 实现参考：`run_topic._prefilter()` + `judge_topic()`；`run_boards._prefilter()` + `judge_llm()`。

## 3. 多维打分与综评 (Multi-Dimensional Scoring + Composite)

留下的候选进入**五维打分**，再由一次综评调用合成 0–10 排序分。**关键原则：综评只做加权合成，绝不回改五个子分。**

### 五个维度（各 0–5）
| 维度 | 含义 | 要点 |
|---|---|---|
| **新鲜度 freshness** | 距发表时间的 sigmoid 衰减 | 代码算出（非 LLM）；见 §4 |
| **可复现性 reproducibility** | 4 子项：开源代码 / 数据可得 / 超参披露 / 环境清晰 | 各 0/0.5/1，带 reason |
| **方法新颖度 novelty** | 4 子项：问题新 / 方法新 / 理论贡献 / 相对增量 | **随论文类型自适应**：综述不因「没提新算法」而被打低；基准看填补的空白 |
| **领域相关性 domain-relevance** | 对我们**具体关注方向**的贴合度 | 阶梯：正中 core 方向≈5；泛化 AI≈4；泛化 CS≈3；边缘 1–2 |
| **权威性 authority** | 机构/作者/发表质量 | 见下方专述；无信号 → **N/A（不评分，绝不给 0）** |

**空签名守卫**：若一次打分所有子项全 0 且无理由 → 判为 LLM 调用失败（HTTP/超时/解析），**抛错、绝不落库/推送伪造的全零分**。

### 综评 composite（0–10，单次调用）
- 输入是**已定的**五个子分，模型只做综合权衡、**不得改动子分**。
- 权重取向：**领域相关性与新颖度通常最重要**；可复现性/权威性为加分；新鲜度次要；**权威性 N/A 不扣分**。
- 输出 `composite`(0–10) + ≤40 字理由（兼作「入选理由」）。失败 → na（排序置底），不伪造。

### ★ 权威性维度详解（最可迁移到「创作者可信度」）
这是最值得迁移到 YouTube 白名单的一维——它把「**谁做的、发在哪**」变成可判断的信号，而**刻意不等于热度/流量**。

**喂给判官的事实（facts）**：
- 从 **PDF 第 1 页**抽取的作者**机构**（正则 + 邮箱域名；失败再用一次便宜 LLM 兜底抽取）。
- **发表/投稿信息**（venue + 状态：已被顶会顶刊接收 / under-review / 仅预印本）。
- 论文方向（标题 + 分类 + 摘要）——因为权威性是**方向内**的权威性。

**打分原则（AUTH_SYS 核心）**：
- 这是**得体的信息披露，不是贬低**；正当合法机构给**公允中档分（≈3 起步）**，不因不熟悉就打低。
- **看全部作者与全部机构，取整组最强信号**——资深/大牛常在**末位（通讯/PI）或中间**，**不要只看第一作者**；任一位置出现该子领域领军人物即算强信号。
- **方向对口的强**才算强：某实验室以计算金融著称 → 其量化论文很权威，哪怕学校综合排名不顶尖。
- **顶会/顶刊已接收**加权更高（NeurIPS/ICML/ICLR/CVPR/ACL/顶刊…）；**"under review" ≠ 已接收**，不据此加满分。
- 对不熟的机构 → 列入 `need_lookup`，用 **OpenAlex**（作品数/被引/强项领域）兜底再问一次。
- **几乎无任何可辨识机构/作者/发表信号 → na=true（不给 0）**。
- 评分参考：顶尖机构/大牛且方向对口≈5；知名机构或方向强≈4；正当普通高校/公司≈3。

> 实现参考：`scorer.score_authority()` / `AUTH_SYS` / `_authority_facts()` / OpenAlex 兜底；`scorer.generate_composite()` / `COMPOSITE_SYS`；空签名守卫 `scorer.is_empty_score()`。

## 4. 新鲜度处理：sigmoid 双轨 + 奠基之作单列 (Freshness)

### Sigmoid 衰减曲线
```
S(t) = 1 + 4.5 / (1 + exp( (ln2 / scale) · (t − center) ))     # t = 距发表天数, S∈[1,5.5]→标定 0–5
```
- **陡峭（steep，默认 center=45, scale=15）**：`S(0)=5, S(45)=3.25, S(90)=1.5`——用于「最新」区，强烈奖励新。
- **平缓（relaxed，center=365, scale=600）**：老经典不被压到 1（如 1125 天的论文仍≈2.3）——用于「经典」区。

### 两轨设计（为什么）
同一条 sigmoid，**换 center/scale 即得两种时间观**：
- 「最新」区用 **steep** → 新鲜度强烈区分，最近的顶上来。
- 「经典」区用 **relaxed** → 经典按**质量**排（见 §5），新鲜度几乎不惩罚老作品。

### 奠基之作单列 (Foundational Callout)
超过某年龄阈值的**验证锚点**（老而定义领域的作品）**单独列成「🌱 奠基之作」，不参与 Top-N 排名**——避免用陡峭新鲜度把 Voyager/ADAS 这类奠基作硬压出榜，同时又完整呈现领域地图。
> 注意：只有**老锚点**进单列；普通老候选仍照常参与排名（steep 自然压低，但凭复现/新颖/权威竞争）。

> 实现参考：`scorer.freshness(center, scale)`；`run_topic._is_foundational()` / `_foundational_rows()`。

## 5. 双区呈现：经典（按质量）+ 最新（按新鲜）(Two-Section Pattern)

最终榜单**分两个互斥区**，让「时间沉淀的重要」与「最新前沿」**都不被对方压垮**：

| 区 | 内容 | freshness 轨 | 排序依据 |
|---|---|---|---|
| **⭐ 经典必读 (Classics)** | 验证种子 | relaxed | **composite 综合质量**（不看新旧） |
| **🆕 最新进展 (Latest)** | 关键词/宽查询命中（去种子） | steep | composite（含强新鲜度） |

- **互斥**：种子只进经典区，**绝不出现在最新区**。
- 若某主题「最新」供给本就稀少（如每类 ~6–7 篇），**如实呈现、不为凑数注水**（诚实的稀缺胜过填充噪声）。
- 单一主题综述场景可退化为「单一 Top-N 榜 + 奠基之作单列」（见 `run_topic`）。

> 实现参考：`run_boards`（A/B 板块 × 经典/最新双区）；`run_topic`（单区 Top-N + 奠基之作）。

## 6. 适配到 YouTube 频道白名单筛选 (Adaptation Notes)

> 以下把每一层**映射**到「创作者白名单」问题，只**指出哪些原则可迁移、如何迁移**，不规定具体清单/参数。领域范围同上：AI / Quant / Math / AI4Science(尤其 Math) / CS exploration。

### 可迁移的原则

**① 三层来源 → 三层找频道**
- **种子锚点** = 公认高质量创作者：研究者本人频道、实验室/公司 CTO/首席、Fields 奖得主讲座、顶级课程作者。以「频道 ID/handle」精确锚定，并做**「校验」**（频道简介/代表作是否与预期人物/主题一致，防同名冒名）。种子豁免「活跃度」门槛（大牛可能更新很慢）。
- **关键词网** = 与种子**频繁互相引用/合作/同框**的频道（连麦、客座、被种子在视频里推荐、共同出现在会议频道）——抓「谱系/圈子」。
- **宽查询** = 领域宽泛搜索，兜住**新兴但优质**的创作者（种子圈子之外的新面孔）。
- *理由同论文*：种子保底、关键词抓血脉、宽查询抓新颖；来源只管「进池」，去留交给判官。

**② LLM 判官 → 频道级 keep/drop**
- 便宜预筛（频道主题标签/近期视频标题像不像目标学科）→ 再一次 LLM 判官：
  「**这个频道是否真正在认真的技术层面做 AI/Quant/Math/AI4Science/CS 前沿？**」→ keep/drop + 子类 + 理由。
- 判官应像论文版一样**明确排除常见误命中**：泛泛「科技资讯/新闻搬运」、纯营销/带货、标题党解说、面向零基础的浅层科普、AI 生成的低质内容农场。
- 喂给判官的「摘要」= 频道简介 + 若干代表视频标题/描述（相当于论文的 title+abstract）。

**③ 权威性维度 → 最重要、最可迁移**
- 用**机构/资历/同行认可**判可信度：作者是否为在职研究者/教授/知名实验室成员、是否有论文/引用记录、是否被领域内公认人物推荐或同台。
- **迁移「取全组最强信号、方向内权威」原则**：一个频道即使订阅不多，若创作者是该子方向的实干者/领军者，仍是强信号。
- **⚠ 要避开的陷阱**：**可信度 ≠ 播放量/订阅数**。论文里我们刻意用 venue/机构而非「热度」——白名单同样**不能用观看量当质量代理**（高播放常来自娱乐化/标题党）。把「流量」和「权威」分开，权威优先。
- 类比「顶会 = 强信号」：可迁移为「在顶级会议/机构做过报告、被顶级课程/研究者引用」= 强信号；「订阅百万但内容浅」≠ 强信号。
- 类比 OpenAlex 兜底：对不熟悉的创作者，可查其学术档案/GitHub/主页做外部核实，再判一次。

**④ 双区呈现 → 「奠基/权威」+「活跃/新兴」**
- **奠基/权威频道**（按质量与可信度排，不看更新频率；含更新缓慢的大牛讲座）+ **活跃/新兴频道**（按活跃度/上升势头排）。
- 「奠基之作单列」→ 经典课程/里程碑讲座**单列**，不与日更频道拼「新鲜度」。
- 供给稀缺时**如实少列**，不为凑满 30–40 注水。

### 不能干净迁移的部分（务必标注）

- **没有 arXiv 式的统一 API**：论文有 arXiv（结构化元数据 + 精确 ID + 摘要 + PDF）。YouTube 无对等的开放学术接口；频道元数据零散、需要不同抓取方式（且本任务是**方法论文档，不写抓取代码**）。
- **「新鲜度」语义变了**：论文是「发表日期」的单调衰减；频道是**更新节奏/持续活跃度**（cadence），不是单一时间戳。sigmoid 那套需重定义为「近 N 月是否稳定产出」，且**对奠基频道要像 relaxed 一样宽容甚至豁免**（经典课程不更新不代表低质）。
- **质量信号更浑浊**：论文有**同行评审的 venue**这一强外部信号；视频**没有同行评审**。可信度需用**代理信号拼图**（创作者学术身份、机构、被引/被推荐、内容技术深度），每个都比「顶会接收」更弱、更需要人工复核。
- **可复现性维度基本不适用**：论文的「开源代码/数据/超参」四子项对频道无直接对应（勉强可类比「是否给出代码/notebook/可跟做的材料」，但远非核心）。
- **单位是「频道」而非「单篇」**：一个频道内容质量参差；判官要基于**代表性作品 + 整体定位**判断，而非单条视频——比论文「一篇一判」更需抽样与整体权衡。
- **判官成本/边界**：频道「摘要」不如论文摘要标准化，判官更易受简介营销话术误导——预筛与理由留证、人工抽查更关键。

### 一句话迁移准则
> **保留**：三层召回（种子保底 + 关系网 + 宽搜）、语义 LLM 判官（keep/drop + 理由）、**权威优先于热度**的可信度打分、双区（权威 × 活跃）+ 奠基单列、诚实稀缺不注水、绝不伪造分数。
> **重定义**：freshness = 活跃节奏；authority = 学术身份/机构/同行认可的代理拼图。
> **舍弃/弱化**：arXiv 式 API 依赖、可复现性四子项、单篇粒度假设。
